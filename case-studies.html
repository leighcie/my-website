Project Case Study: Redesigning the Renewals Process for the State Insurance Marketplace

Overview: I spearheaded the overhaul of the renewals process for the state insurance marketplace to enhance efficiency and user experience.

Problem Statement

Context: The renewals process required manual initiation and continuous monitoring, leading to substantial system downtimesâ€”typically exceeding 8 calendar days.

Specific Problems: We aimed to automate and expedite the renewals process to identify eligible customers, verify their eligibility, offer automatic and option-based renewal system while minimizing downtime.

Solution

Approach: We mapped out existing processes and pain points, then evaluated our newly designed service-oriented architecture for enhancements.

Steps:

  • Conducted a detailed analysis of the manual process.
  • Mapped the current workflow and pinpointed inefficiencies.
  • Designed an automated solution leveraging our enhanced service-oriented architecture.
  • Collaborated with stakeholders to gather requirements and align on goals.
  • Developed, tested, and implemented the new process in an AWS environment.
  • Monitored and refined the solution post-implementation.

Technologies and Tools: Agile methodologies (Scrum), service-oriented architecture, AWS, Java, Spring Boot, Jira (work management), Confluence (documentation), PowerPoint (presentations), Miro (process mapping), Slack (team collaboration).

Results

Outcomes: The new automated process successfully processed 87% of renewals within 24 hours.

KPIs: We set a goal to process 80% of renewals within 3 days.

Impact: This significantly reduced downtime, allowing support staff and customers more time to make insurance selections. Despite concerns from a recent data migration, we surpassed our target renewal rate.

Learnings and Reflections

Challenges Faced: The process involved multiple teams, requiring intensive collaboration under tight deadlines. We encountered gaps in requirements and data errors that needed quick resolution.

Key Learnings: Success relies on shared ownership across teams. Effective collaboration, documentation of edge cases, and streamlined error handling can set teams up for future successes.

Future Improvements: While the project was a success, future efforts could focus on optimizing eligibility determination processes for even greater efficiency.

Project Case Study: Enhancing Data Reconciliation Corrections for the Mortgage Industry

Overview: I led the enhancement of Mintrak2, a SaaS platform used by servicers and sub-servicers in the mortgage industry, to improve efficiency and user satisfaction in data reconciliation with the MERS registry.

Problem Statement

Context: Mintrak2 enabled customers to upload their datasets alongside the MERS dataset for comparison and discrepancy identification. However, customers frequently encountered the same discrepancies each month due to delayed system updates.

Specific Problems: We sought to streamline the reconciliation process by helping users prioritize and address discrepancies effectively and enable quicker system updates to reduce recurring issues.

Solution

Approach: We analyzed the existing processes and customer feedback, then identified and implemented targeted improvements to enhance the user experience.

Steps:

  • Conducted a detailed analysis of customer feedback and system usage data.
  • Mapped the current workflow to identify inefficiencies and recurring pain points.
  • Implemented discrepancy sorting to allow users to prioritize and address older discrepancies first.
  • Developed a custom export feature enabling users to quickly generate updated datasets for their IT teams.
  • Collaborated with stakeholders to gather requirements and align on goals.
  • Tested and implemented the new features.
  • Monitored and refined the solution post-implementation.

Technologies and Tools: Agile methodologies (Scrum), Jira (work management), Microsoft Word and Adobe PDF (documentation), PowerPoint (presentations), Visio (process mapping), Microsoft Teams (team collaboration).

Results

Outcomes: The new features led to significantly happier clients who experienced fewer recurring discrepancies.

KPIs: Target: Reduce recurring discrepancies by 50% within 6 months. Achieved: 60% reduction in recurring discrepancies within 3 months.

Impact: The streamlined reconciliation process improved user satisfaction, with MERS recommending these features to Mintrak2 users due to noticeable improvements in data comparisons.

Learnings and Reflections

Challenges Faced: Integrating new features without cluttering the page and ensuring they were intuitive for users. For customers servicing loans in-house and subservicing others, we needed a way to export data by servicer. Although these data elements existed, inaccuracies in the servicer or subservicer information required us to create separate files for each party based on the customer's remediation.

Key Learnings: Detailed and iterative testing is essential, especially when dealing with custom file types. Fixing one issue often introduced new ones, highlighting the need for a robust testing plan. Effective communication and collaboration across teams ensured that everyone was aligned, and stakeholders received the correct information. Additionally, leveraging existing UI elements and keeping the design intuitive were critical for user adoption and satisfaction.

Future Improvements: A more thorough and consistent testing plan for custom file types to prevent new bugs from being introduced when fixing existing ones.
